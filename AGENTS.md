Context: this repo is a **project folder** for an OpenFrameworks app. OpenFrameworks (OF) itself is installed separately (e.g. `~/Documents/OpenFrameworks/` with `apps/`, `libs/`, `addons/`, `projectGenerator/`).

## Mental model (if you know Unity + JS)

| Concept | Unity | JavaScript / Node | OpenFrameworks |
|--------|--------|-------------------|----------------|
| **Runtime** | Player loop + scenes | Event loop (Node) / browser frame | **`update()` + `draw()`** every frame (like `Update` + `OnGUI`/render pass, but you own the window) |
| **Entry** | `MonoBehaviour` on GameObjects | `index.js`, `main` | **`ofApp`**: your `ofBaseApp` subclass in `src/` |
| **Assets** | `Assets/` | `public/`, imports | **`bin/data/`** — images, shaders, sounds; load with `ofLoadImage`, `ofSoundPlayer`, etc. |
| **Packages** | Package Manager / UPM | `npm` | **`addons/`** — copy or clone into `openFrameworks/addons/`, enable in Project Generator or CMake |
| **Build** | Editor builds | `npm run build` | **CMake** (OF 0.11+) or **Xcode** (macOS) / **Visual Studio** (Windows) — generated by **Project Generator** |
| **Memory** | GC (managed) | GC | **Manual-ish**: stack vs heap; smart pointers where OF uses them; avoid leaking `new` without `delete` |

OF is **not** a full game engine: no scene graph, no physics out of the box (you add addons), no editor. It is a **C++ toolkit** for creative coding: 2D/3D graphics, audio, video, OSC, serial, etc.

## Minimal C++ you need for OF

- **Headers (`.h`) vs implementation (`.cpp`)**: Declarations often live in `.h`; definitions in `.cpp`. Your app is usually `ofApp.h` + `ofApp.cpp` + `main.cpp`.
- **`#include`**: Pulls in declarations. OF uses `"ofMain.h"` and your headers with quotes.
- **Namespaces**: OF types are mostly in global namespace or `glm::` for vectors from GLM (OF uses GLM for math in newer APIs).
- **References `&` vs pointers `*`**: OF APIs often take references (`ofVec2f &`) or pointers for optional/nullable things. Follow examples literally.
- **`std::vector`**: Dynamic arrays — like `Array` in JS, fixed element type.
- **No garbage collection**: Prefer stack objects (`ofImage img;`) and OF helpers; if you `new`, you must `delete` (or use smart pointers consistently).

## Application lifecycle

Rough order each frame:

1. **Startup**: `main()` creates `ofApp` and runs the OF window loop.
2. **`setup()`** — once after the window exists. Load fonts, allocate FBOs, open serial, set `ofSetFrameRate`, etc.
3. **Per frame** (typical):
   - **`update()`** — simulation, input handling, animation, audio analysis. **No drawing here** (convention).
   - **`draw()`** — all drawing (`ofBackground`, `ofDrawRectangle`, meshes, cameras).
4. **Events** (as needed): `keyPressed`, `mouseMoved`, `windowResized`, etc.
5. **Shutdown**: `exit()` if you override it — release resources (close files, etc.).

Pseudo-flow:

```text
setup() once
loop:
  update()
  draw()
  (OS delivers input events between frames)
```

## Kickstart workflow (new project)

1. Install **OpenFrameworks** from [openframeworks.cc](https://openframeworks.cc/) for your OS.
2. Run **`projectGenerator`** (in the OF download): set project path (this folder or a subfolder), name, platform, optional addons.
3. Open the generated **Xcode** / **Visual Studio** / **Code::Blocks** project, or use **CMake** if that’s your OF version’s workflow.
4. Build and run; confirm a window opens.
5. Put assets in **`bin/data/`** relative to the app executable.

**Tip:** Keep this repo as the app root **or** put the generated app inside `LQCAA/` — be consistent so `bin/data/` paths stay predictable.

## Coding habits that help

- **Small `draw()`**: Do heavy work in `update()`; keep `draw()` for state + rendering.
- **Use `ofVec`, `ofRectangle`, `ofColor`** before reinventing math.
- **Frame rate**: `ofSetVerticalSync(true)` to reduce tearing; cap with `ofSetFrameRate(60)` if needed.
- **Resolution / retina**: Be aware of `ofGetWidth()` / `ofGetHeight()` vs framebuffer pixel size on HiDPI — test fullscreen and scaling.
- **Addons**: Prefer maintained addons; read each addon’s `README` for dependencies and extra compile steps.

## Debugging

- **macOS**: Xcode debugger, breakpoints in `update`/`draw`.
- **Logging**: `ofLogNotice("tag") << "value " << x;` — avoid `std::cout` alone in tutorials if OF log levels matter.
- **Crashing on load**: Often missing file in `bin/data/` or wrong path — check working directory is next to the executable.

## When to reach for what

- **2D shapes / images**: `ofGraphics`, `ofImage`, `ofFbo` for offscreen passes.
- **3D**: `ofEasyCam`, `ofMesh`, `ofLight` — OpenGL-style pipeline.
- **Shaders**: `ofShader` + `.vert` / `.frag` in `bin/data/`.
- **Audio in**: `ofSoundStream` for low-latency; `ofSoundPlayer` for playback.
- **OSC / MIDI / Serial**: dedicated addons and core examples.

## Building in Cursor (without opening Xcode as your daily driver)

**Yes.** Cursor has no built-in C++ compiler; you build from the **integrated terminal** (or tasks) using the same toolchain Xcode would use.

- **macOS still needs a compiler**: Install **Xcode** or **Xcode Command Line Tools** (`xcode-select --install`). You are only avoiding the Xcode *app*, not the SDK/clang.
- **Typical options:**
  - **CMake** (common for OF 0.11+): `cmake -S . -B build && cmake --build build` (exact flags depend on your generated project).
  - **Existing `.xcodeproj`**: `xcodebuild -scheme YourApp -configuration Debug` from the project directory.
- **In Cursor / VS Code**, useful extensions:
  - **C/C++** (Microsoft) — IntelliSense, debugging integration.
  - **CMake Tools** — configure/build if the project is CMake-based.
  - **clangd** — alternative language server (don’t enable both clangd and MSVC IntelliSense on the same folder without tuning).
  - **CodeLLDB** — debugging native binaries from the editor on macOS.

Point include paths at your **OpenFrameworks** install (`libs/openFrameworks`, etc.) so navigation and completion work; Project Generator / CMake usually defines these for the build, but the editor may need `.vscode/c_cpp_properties.json` or CMake Tools to match.

## OpenSpec (structured workflow alongside OF)

**OpenSpec** ([Fission-AI/OpenSpec](https://github.com/Fission-AI/OpenSpec)) is **stack-agnostic**: it adds a spec-driven workflow (proposals, tasks, archived changes) for AI-assisted development. It does **not** compile C++; it organizes *how* you specify and implement features. It pairs well with this repo’s **`AGENTS.md`**.

- **Requirements**: Node.js **20.19+** (see project docs).
- **Install CLI**: `npm install -g @fission-ai/openspec@latest`
- **Initialize in this repo**: `openspec init` (from the project root after the OF app exists or alongside it — keep one clear root for both).
- **Workflow**: Use slash commands like `/opsx:propose` and `/opsx:apply` as documented in OpenSpec; run `openspec update` after upgrades.

Use OpenSpec for **what to build** (specs, tasks); use OF + CMake/Xcode for **how it compiles**.
